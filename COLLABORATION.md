## 一、协作约定（重要）

1. **主线是 `main`**：`main` 永远是「最新、最全、可部署」的那一份。Railway 部署盯 `main`。
2. **开工前先同步**：`git fetch` + 基于最新 `main`（或共享开发分支）再动手；动手前 `git pull`。
3. **改完合回主线**：用 PR 或直接合并把改动并回 `main`，别让改动长期散在私有分支。
4. **保持函数签名稳定**：`npc.py` 暴露给 `bot.py` 的函数（`make_npcs / speak / vote_decision /
   seer_check_target / wolf_kill_target / witch_night_action / wolf_chat / last_word`）签名尽量别改；
   要改就同步改 `bot.py` 调用处，并在下方「改动记录」里写明。
5. **小步提交、写清楚 commit**：commit message 说明「为什么改」，方便对方回溯。
6. **改完自检**：`python -m py_compile bot.py npc.py llm.py config.py game/*.py characters.py knowledge.py`
   能过；尽量加一个离线 smoke test（不连网也能验逻辑）。

## 二、当前架构速览

| 文件 | 职责 |
|---|---|
| `bot.py` | Discord 交互层：slash 命令、大厅、**面板模式**、夜晚/白天/投票 UI、整局流程 |
| `game/` | 纯游戏逻辑（与 Discord 解耦）：`roles` 角色板子、`player` 玩家、`state` 状态机/结算/胜负 |
| `npc.py` | AI NPC：补位、**决策（规则）**、**发言（LLM+人设）**、私人记忆 `_MEMORY` |
| `characters.py` | 有完整人设的「角色 NPC」登记表（当前：**Theo、闻人幸**），补位时优先入座 |
| `knowledge.py` | 狼人杀**常识手册**，注入进 NPC 发言提示，让 AI 打牌更懂套路 |
| `llm.py` | LLM 中转站客户端：调用 + 重试 + **全局限速闸** + 开机自检 + 报错翻译 |
| `config.py` | 所有环境变量配置（板子人数、各阶段时限、LLM 超时/限速间隔等） |

**板子**：6 人 = 🐺狼 ×2 / 🔮预言家 ×1 / 🧪女巫 ×1 / 👤平民 ×2（`game/roles.py` 自适应）。

## 三、重要注意事项 / 已经踩过的坑 ⚠️

1. **API 限流（429）是头号敌人**。中转站多为「每分钟」限流。当前策略：
   - **决策走规则（不花 API）**：验/刀/救毒/投票都是 `npc.py` 里的轻量规则；
   - **发言走 LLM（花 API）**：`speak / wolf_chat / last_word` 才调模型，这是最像人、最值得花的部分；
   - **全局限速闸** `llm._throttle()`：所有调用串行 + 强制最小间隔（`config.LLM_MIN_INTERVAL_SECONDS`）。
   - ❗ **别再把决策改成「每个决策都调一次 LLM」**，那会瞬间把额度打爆（之前就是这么爆的）。
2. **投票 UI 必须用「公开按钮 → 私有下拉选单」模式**。
   把 `discord.ui.Select` 直接挂在**公开频道消息**上会「交互失败」；
   夜晚预言家/狼/女巫和投票现在都走「公开按钮点开 → ephemeral 私有选单」这条已验证可用的链路。
3. **面板模式 + 统一禁言**：游戏频道里玩家默认不能直接打字（消息会被删），一切走面板按钮；
   轮到发言时点按钮弹输入框。改 UI 时注意别破坏这个前提。
4. **全程匿名**：公开场合只显示**座位号**（如「3号」），不暴露是谁、不 @ 人（提醒走私信 `dm_user`）。
5. **NPC 决策函数是 `async`**（即使内部只有规则），因为 `bot.py` 用 `await` 调它们——改时保持 `async`。
6. **狼可自刀**：选刀菜单已放开对狼/自己的限制（骗药/搏信任战术）；夜晚结算无需特殊处理。
7. **人设注入统一走 `_persona_clause()`**：通用 NPC 是一句性格，角色 NPC 是整段设定，别绕开它。
8. **多人同时改同一文件**时，`npc.py` 最容易撞——动手前 fetch，改完尽快合回主线。
